# Project Overlay — Dontpad

## Objetivo

Este diretório contém as regras e fatos **específicos do projeto Dontpad**.

## Arquivos principais

- `context.md`: baseline factual do produto, stack, APIs e convenções.
- `rules.md`: prioridades e regras específicas de domínio/técnicas.
- `workflow-overrides.md`: restrições operacionais específicas da execução.

## Ordem de leitura recomendada

1. `rules.md`
2. `context.md`
3. `workflow-overrides.md`

## Regras de uso

- Este diretório complementa `core/*` com informações do Dontpad.
- Em caso de conflito com `core/*`, aplicar a precedência definida em `_docs/ia-context/README.md`.
